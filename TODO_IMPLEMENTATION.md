# Echo 系统深度优化与待办功能清单 (TODO Implementation)

本文档记录了当前系统中为了性能和体积进行的“简化实现”逻辑，以及未来实现更高阶（SillyTavern 级别）功能的演进方向。

---

## 1. 提示词引擎 (Prompt Engine) 演进

### 递归扫描优化 (Recursive Scanning)
- **当前现状**：`scanActiveWorldInfo` 使用了固定的 2 层递归深度（A -> B -> C），且扫描时简单的拼接了所有内容。
- **优化方向**：
    - **动态深度**：允许在全局设置中配置递归层数。
    - **循环检测**：目前的实现虽然有深度限制，但如果条目 A 引用 B，B 引用 A，会造成不必要的 Token 浪费。需要增加 `Set` 来追踪已扫描条目。
    - **权重继承**：被递归唤醒的条目是否应该继承父条目的权重？

### 正则表达式支持 (Regex Keys)
- **当前现状**：目前世界书关键词仅支持普通字符串（带单词边界匹配）。
- **优化方向**：支持完整的 JavaScript 正则表达式作为 Key，例如 `/^#inventory/gi`。

## 2. Token 管理与截断算法

### 精确词元化 (Tokenizer)
- **当前现状**：`tokenCounter.ts` 使用启发式算法（中 1.2, 英 3.5）。这在大多数情况下误差在 10% 以内，但在极端极端长文本下会有累积误差。
- **优化方向**：
    - **Web Worker Tiktoken**：引入轻量级 `tiktoken` 并在 Web Worker 中运行，避免阻塞主线程。
    - **模型特定 Tokenizer**：根据 Provider 自动切换 OpenAI/Anthropic/Llama 对应的词元算法。

### 动态预算分配 (Smart Budgeting)
- **当前现状**：世界书固定占用 35% 预算。
- **优化方向**：允许用户自定义“世界书预算比例”和“历史记录最小保留量”。

---

## 3. 语音合成 (TTS) 系统增强

### Edge TTS 原生集成
- **当前现状**：`ttsService.ts` 中的 `edge` 模式目前回退到了浏览器原生语音。
- **优化方向**：集成 `edge-tts` 的 WebSocket 协议，或者在后端 `echo-storage` 增加中转接口，以获得更高质量且免费的微软语音。

### 语音流式预加载 (Streaming TTS)
- **当前现状**：等待 AI 回复完整结束后才触发语音。
- **优化方向**：实现流式语音。当 AI 输出第一句完整话语（匹配到句号/问号）时，立即发起 TTS 请求并播放，显著降低等待感。

## 4. 插件与 API 兼容性

### Assistant Prefill (引导语注入)
- **当前现状**：尚未实现。
- **优化方向**：在 API 请求的最后一条消息之后，允许用户自定义一个可选的 `assistant` 角色起始词（如 `"` 或 `*我看着你，微笑着说：*`），强制模型进入特定叙事视角。

### 自动模型列表获取 (Model Discovery)
- **当前现状**：模型名需要手动输入。
- **优化方向**：增加“刷新模型”按钮，请求 `/v1/models` 接口并自动填充下拉菜单。

---

## 5. UI/UX 体验

### 多模态渲染优化
- **当前现状**：图片以 Base64 直接存储在聊天记录中。
- **优化方向**：对于长程存档，Base64 会导致数据库体积剧增。建议将图片存储到 `IndexedDB` 并在消息中只记录 `blobId`。
